## 🔴 KRİTİK (3 adet)

### K-NEW1 — Express 4 Async Handler'larda try-catch Yok
- **Dosyalar:** `routes/depolar.js`, `routes/sayimlar.js`, `routes/urunler.js`, `middleware/yetkiGuard.js`
- **Etkilenen:** 15+ async handler
- **Risk:** `await pool.execute()` hata firlatirsa Node.js process coker. Express 4 async hatalari otomatik yakalamaz.
- **Cozum:** Tum async handler'lara try-catch ekle veya express-async-errors paketi kullan.

### K-NEW2 — isletme_ids Yetki Bypass
- **Dosya:** `routes/sayimlar.js:48`
- **Risk:** `GET /sayimlar?isletme_ids=1,5,9` gonderildiginde sadece ilk ID (1) icin yetki kontrolu yapiliyor. Diger ID'ler (5, 9) kontrol edilmiyor.
- **Etki:** Yetkisiz isletmelerin sayimlari gorulebilir.
- **Cozum:** `isletme_ids.split(',')` ile tum ID'ler icin yetki kontrolu yapilmali.

### K-NEW3 — yetkiGuard Null Yetkiler Crash
- **Dosya:** `middleware/yetkiGuard.js:33`
- **Risk:** DB'den `yetkiler: null` gelirse `yetkiler[kategori]` satirinda `TypeError` firlatir, process coker.
- **Cozum:** `const yetkiler = rows[0].yetkiler || {}` seklinde null check ekle.

## 🟠 YUKSEK (6 adet)

### Y-NEW1 — err.message Sizintisi (4 endpoint)
- **Dosyalar:**
  - `routes/isletmeler.js:176` (PUT /:id/restore)
  - `routes/kullanicilar.js:398` (PUT /:id/restore)
  - `routes/profil.js:64` (GET /stats)
  - `routes/profil.js:80` (PUT /ayarlar)
- **Risk:** Internal hata detaylari client'a donuyor.
- **Cozum:** `res.status(500).json({ hata: err.message })` → `console.error(...); res.status(500).json({ hata: 'Sunucu hatasi.' })`

### Y-NEW2 — Barkod Race Condition (TOCTOU)
- **Dosya:** `routes/urunler.js:67-83`
- **Risk:** Barkod benzersizlik kontrolu SELECT ile yapiliyor, ardindan INSERT. Iki eszamanli istek ayni barkodu ekleyebilir.
- **Cozum:** DB'de UNIQUE constraint veya transaction kullan.

### Y-NEW3 — Toplama Race Condition
- **Dosya:** `routes/sayimlar.js:504-589`
- **Risk:** Toplama islemi (sayimlari birlestirme) transaction olmadan yapiliyor. Eszamanli istek veri tutarsizligina yol acar.
- **Cozum:** `pool.getConnection()` + `connection.beginTransaction()` ile transaction icinde yap.

### Y-NEW4 — Rol Silme Race Condition
- **Dosya:** `routes/roller.js:148-203`
- **Risk:** Kullanici reassign + rol delete ayri sorgularda. Process crash olursa veri tutarsiz kalir.
- **Cozum:** Transaction kullan.

### Y-NEW5 — parseFloat(null) NaN Bug
- **Dosya:** `routes/sayimlar.js:552`
- **Risk:** `toplamMap[k.urun_id].miktar += parseFloat(k.miktar)` — miktar null ise NaN uretir, tum toplama sonuclari bozulur.
- **Cozum:** `parseFloat(k.miktar) || 0`

### Y-NEW6 — Kullanici Olusturmada Sifre Validasyonu Yok
- **Dosya:** `routes/kullanicilar.js:155`
- **Risk:** POST ile kullanici olusturulurken sifre minimum uzunluk kontrolu yok. 1 karakterlik sifre kabul edilir.
- **Cozum:** `if (!sifre || sifre.length < 6) return res.status(400).json({ hata: 'Sifre en az 6 karakter olmali.' })`
